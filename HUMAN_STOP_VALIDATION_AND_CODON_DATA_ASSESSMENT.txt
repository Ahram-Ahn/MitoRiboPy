4) codon_data.py -> offset_analysis flow assessment
Question: Is a required assignment missing?

Current flow is logically correct for the pipeline path:
- codon_data.py provides annotation columns:
  - transcript, l_tr, l_utr5, l_cds, l_utr3
- main.py derives required offset columns before offset analysis:
  - start_codon = l_utr5
  - stop_codon = l_tr - l_utr3 - 3
- offset_analysis.py then uses:
  - transcript
  - start_codon / stop_codon

So, for normal pipeline usage, no missing assignment exists.

Important nuance:
- If someone calls offset_analysis.compute_offsets(...) directly with raw human_annotation_df from codon_data.py (without going through main.py), start_codon and stop_codon would indeed be missing.
- That may be what another AI was referring to.

Practical recommendation:
- Keep current behavior (it is correct in pipeline execution), or add a helper function (e.g., prepare_annotation_df) to make this dependency explicit and avoid misuse outside main.py.

5) Additional biological caution (not a coding bug)
- human_annotation_df uses combined entries (ATP86, ND4L4) by design.
- These are valid if your FASTA/transcript naming and coordinates are meant to be combined ORFs, which they are in your current human FASTA.
- If you switch to gene-split references later, these annotation rows should be updated accordingly.
