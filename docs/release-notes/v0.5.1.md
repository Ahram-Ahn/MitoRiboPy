# v0.5.1 — `mitoribopy rnaseq` gets a richer WT-vs-X plot set, publication styling, and `--base-sample` / `--compare-sample` aliases

`mitoribopy rnaseq` now emits a fuller comparison panel rendered at publication quality (Okabe-Ito colour-blind-safe palette, sans-serif fonts, 300 dpi PNG + editable SVG sidecar, all data points shown on bar plots), and exposes the contrast under names that mirror the rpf config. Same inputs, more — and better — figures.

## Added: `--base-sample` / `--compare-sample` CLI aliases

The reference / comparison conditions of the rnaseq contrast are now reachable under two names:

| Canonical (new) | Legacy (still works) |
|---|---|
| `--base-sample NAME`    | `--condition-a NAME` |
| `--compare-sample NAME` | `--condition-b NAME` |

Why the new spelling: the `rpf` config block already has a `base_sample` key (it picks the reference for codon-correlation plots), and reusing the same word across stages keeps the YAML / CLI surface consistent. The legacy `--condition-a` / `--condition-b` flags are unchanged — pick whichever spelling reads better. Passing both an alias and the legacy form simultaneously is allowed only when the two values agree (a mismatch exits with code 2 and a clear error).

In YAML, the matching keys are `base_sample:` / `compare_sample:` (and the legacy `condition_a:` / `condition_b:` keep working). In `run_settings.json` we now also record `base_sample` / `compare_sample` alongside the legacy `condition_a` / `condition_b` so downstream tooling can pick either field.

## Added: four new comparison plots

`<output>/rnaseq/plots/` gains four figures. Plot titles include the contrast label (`<base> vs <compare>`) so reviewers know the direction at a glance.

| Plot | What it shows | Available in |
|---|---|---|
| `de_volcano_mrna.png` | mRNA DE volcano: `log2FoldChange` (x) vs `-log10(padj)` (y); points coloured **red** (sig up), **blue** (sig down), **grey** (n.s.); threshold guides at `padj < 0.05` and `|L2FC| ≥ 1`; every gene labelled. | both flows |
| `de_volcano_rpf.png` | Same shape, driven by a second pyDESeq2 fit on the **Ribo-seq subset**. Lets reviewers see at a glance whether a TE shift is driven at the footprint level, the mRNA level, or both. | default flow only (needs the per-sample Ribo counts the default flow already has on hand) |
| `te_compare_scatter.png` | Per-gene mean `log2(TE)` in the base condition (x) vs the compare condition (y), with the identity `y = x` line drawn for reference. Genes off the diagonal mark condition-specific TE differences. | both flows (needs `--condition-map` + `--base-sample` + `--compare-sample`) |
| `te_log2fc_bar.png` | Sorted bar of `log2(mean_TE_compare / mean_TE_base)` per gene; bars above zero mean TE up in the compare condition. The fastest-readable summary of TE direction. | both flows (same conditions as above) |

The existing six plots (`mrna_vs_rpf`, `delta_te_volcano`, `ma`, `te_bar_by_condition`, `te_heatmap`, `sample_pca`) are unchanged in shape, but now wear the `<base> vs <compare>` contrast label in their titles when both flags are set.

## Added: `rpf_de_table.tsv`

The default flow now runs `pyDESeq2` **twice** — once on the RNA subset (writes the existing `de_table.tsv`) and once on the Ribo-seq subset (writes the new `rpf_de_table.tsv`). The second fit feeds `de_volcano_rpf.png`. When the Ribo subset has fewer than two condition levels (e.g. you only passed `--ribo-fastq` for one condition), the second fit is **skipped with a stderr WARNING** and the RPF volcano is omitted; the run still produces the mRNA volcano and every TE comparison plot.

Both DE tables share the canonical DESeq2 schema (`gene_id` / `baseMean` / `log2FoldChange` / `padj`), so any external tooling that already consumed `de_table.tsv` can read `rpf_de_table.tsv` with no schema work.

## How to drive the new flow

```bash
mitoribopy rnaseq \
  --rna-fastq input_data/rna_seq/ \
  --ribo-fastq input_data/ribo_seq/ \
  --reference-fasta refs/human-mt-mRNA.fasta \
  --gene-id-convention bare \
  --condition-map samples.tsv \
  --base-sample WT \
  --compare-sample KO \
  --output results/rnaseq/
```

Or in YAML:

```yaml
# rnaseq_config.yaml -- mitoribopy rnaseq --config rnaseq_config.yaml
rna_fastq:  [input_data/rna_seq]
ribo_fastq: [input_data/ribo_seq]
reference_fasta: refs/human-mt-mRNA.fasta
gene_id_convention: bare
condition_map: samples.tsv
base_sample: WT
compare_sample: KO
```

## Changed: publication-style refresh of every rnaseq plot

Every figure under `<output>/rnaseq/plots/` was rewritten to meet a consistent publication-grade look. Same inputs, same file names, but the renderer now applies a shared rc-context across the whole module:

- **Okabe-Ito colour-blind-safe palette.** Up = vermillion, down = blue, n.s. = light grey across every volcano / MA / TE plot, so the colour reads consistently across figures.
- **Sans-serif typography (Arial / Helvetica fallback).** Top + right spines hidden. Minor ticks visible. Subtle axes guides at zero, dashed threshold guides at `padj < 0.05` and `|L2FC| ≥ 1`.
- **300 DPI PNG + editable SVG sidecar** for every plot. SVG keeps text as text (`svg.fonttype = none`) so figures land in Illustrator without becoming path soup.
- **White-bbox labels with leader lines.** Gene labels on volcanos / scatters carry a translucent white bbox so they stay legible when crossing other elements; a thin leader line keeps the point-to-label correspondence unambiguous on dense plots.
- **Smart 8-direction label placer.** A small greedy algorithm (no `adjustText` dependency) tries 8 candidate positions per label and picks the one with minimum overlap against already-placed labels, prefering above-the-point. Handles the ~13-mt-mRNA universe cleanly; pass `annotate=False` for whole-transcriptome inputs.
- **Stat boxes** on the volcanos and the TE compare scatter — `genes plotted: N`, `up:`, `down:`, `n.s.:`, plus Pearson r on the scatter — read at a glance from the figure alone.
- **Per-replicate dots overlaid on `te_bar_by_condition`.** Bars now show mean ± SE *and* every replicate as a black dot jittered along x within the bar's footprint, so reviewers see the variance behind every mean. The publication-grade rule: bars without dots hide the sample size.
- **Condition strip on `te_heatmap`.** A coloured band above the columns spells the condition assignment (`KO` / `RESC` / `WT` etc.), matching the bar-plot palette. Cell-value annotations get auto-contrast (white on saturated, black on faint).
- **Identity line + Pearson r on `te_compare_scatter`.** Per-gene replicate values are also drawn as a faint cloud behind the gene-mean dots so the within-group spread is visible.
- **Tip-labelled bars on `te_log2fc_bar`** with the numeric `log2FC` printed at each bar's tip and a direction-coded legend (TE up vs TE down in compare condition).
- **Quadrant captions on `mrna_vs_rpf`.** Faint "co-regulated up / buffered up / co-regulated down / buffered down" tags in each corner so a reviewer who lands on the figure alone reads the four-quadrant biology immediately. Points coloured by quadrant.

## No breaking changes

- Existing configs / scripts that use `--condition-a` / `--condition-b` (or `condition_a:` / `condition_b:` in YAML) keep working unchanged.
- The default plot set is a strict superset of v0.5.0; nothing was renamed or removed.
- The new SVG sidecars sit alongside the existing PNGs (same stem, `.svg` suffix). Downstream tooling that scans for `*.png` is unaffected.
- `rpf_de_table.tsv` is **new**; downstream tooling that scanned `<output>/rnaseq/` for `de_table.tsv` only is unaffected.
