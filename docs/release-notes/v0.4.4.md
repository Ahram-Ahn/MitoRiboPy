# v0.4.4 — Output layout cleanup, per-site correlation, IGV tracks, FASTQ subsampling

This release fixes the silent `cor_plot` skip bug, flattens redundant `p/` / `a/` subfolders, separates CSVs from plots in the offset-diagnostics dir, adds IGV-compatible BedGraph exports, and ships a FASTQ-subsampling sibling to the BED subsampler.

## Highlights

### Bug fix — `cor_plot: true` with `base_sample` set produced no output

The `run_downstream_modules` gate previously had three branches that silently fell into `skipped_modules.append("codon correlation")` with no log message. When `cor_plot: true` was set in YAML and a valid `base_sample` was provided, a missing combined-offsets table or a sample-name mismatch would skip the step without surfacing the reason.

In v0.4.4 every skip path emits an explicit `[PIPELINE]` log line with the cause:

- `cor_plot is false; skipping codon correlation.`
- `cor_plot is true but no --base_sample was set; skipping codon correlation.`
- `cor_plot requested but no offsets were selected; skipping codon correlation.`
- `base_sample '<x>' is not in the sample list <samples>. Skipping codon correlation.`

### Per-site codon correlation

Codon correlation now produces a separate output folder per requested site (`--analysis_sites both` → both `codon_correlation/p_site/` and `codon_correlation/a_site/`). The function reads `<sample>/codon_usage/p_site_codon_usage_total.csv` for the P-site run and `a_site_codon_usage_total.csv` for the A-site run, with the existing stop-codon override (P-site stop rows are substituted with the A-site value) preserved on the P-site track.

### Flat output layout — drop redundant `p/` / `a/` subfolders

The legacy nested layout duplicated work: `translation_profile/p/<sample>/codon_usage/codon_usage_total.csv` and `translation_profile/a/<sample>/codon_usage/codon_usage_total.csv` had different file contents but identical filenames; the same `footprint_density/<transcript>_footprint_density.csv` (already containing both A_site and P_site columns) was bit-identical between the two parent folders.

v0.4.4 collapses to a single per-sample folder with the site encoded in filename prefixes:

```
translation_profile/<sample>/
  codon_usage/
    p_site_codon_usage_total.csv          # P-site, stop-codon-masked
    p_site_codon_usage_<transcript>.csv
    a_site_codon_usage_total.csv          # A-site, no stop mask
    a_site_codon_usage_<transcript>.csv
  footprint_density/
    <transcript>_footprint_density.csv    # written once
    <transcript>_p_site_depth.png         # per requested site
    <transcript>_a_site_depth.png
  translating_frame/
    p_site_frame_usage_total.csv
    a_site_frame_usage_total.csv
    ...

coverage_profile_plots/
  p_site_density_rpm/<transcript>.png     # site_density_* renamed with site prefix
  p_site_density_raw/...
  p_site_density_rpm_codon/...
  p_site_density_rpm_frame/...
  a_site_density_*/...                    # mirror folders for A-site
  read_coverage_rpm/<transcript>.png      # gated by --read_coverage_rpm
  read_coverage_raw/<transcript>.png      # gated by --read_coverage_raw
```

### `plots_and_csv/` → `offset_diagnostics/{csv,plots}/`

The default offset-output directory is renamed from `plots_and_csv/` to `offset_diagnostics/` and CSVs are split from PNG/SVG plots into sibling folders. Override the parent name with `--plot_dir`.

```
offset_diagnostics/
  csv/
    offset_<align>.csv
    p_site_offsets_<align>.csv
    read_length_summary.csv
    per_sample_offset/                    # renamed from per_sample/
      <sample>/
        offset_<align>.csv
        p_site_offsets_<align>.csv
        offset_applied.csv                # NEW — exact offsets row applied
                                          #       to <sample> downstream
  plots/
    offset_drift_<align>.svg
    offset_*.svg
    unfiltered_heatmap_*.svg
    <sample>_read_length_distribution.svg
```

The new `offset_applied.csv` makes it possible to confirm from disk that per-sample offset selection (`--offset_mode per_sample`, the default) was honoured by the downstream translation-profile and coverage-profile steps. Pure diagnostic; no behaviour change.

### New CLI flags

| Flag | Default | Effect |
|------|---------|--------|
| `--read_coverage_raw` / `--no-read_coverage_raw` | on | Toggle `coverage_profile_plots/read_coverage_raw[_codon]/` folders. |
| `--read_coverage_rpm` / `--no-read_coverage_rpm` | on | Toggle `coverage_profile_plots/read_coverage_rpm[_codon]/` folders. |
| `--igv_export` / `--no-igv_export` | off | Write per-sample BedGraph tracks under `igv_tracks/<sample>/<sample>_{p_site,a_site}.bedgraph`. P-site tracks use forest green, A-site dark orange (set via the BedGraph `track color=` header). Loads directly into IGV alongside the FASTA the BEDs were aligned to. |

### FASTQ subsampler

A sibling of the existing BED subsampler — reservoir samples 4-line FASTQ records:

```bash
python -m mitoribopy.tools.subsample_fastq \
    --input  raw/sample.fastq.gz \
    --output /tmp/sample_subsampled.fastq.gz \
    --n 200000 \
    --seed 42
```

Auto-detects gzip via the `.gz` suffix on either end. The first record header must start with `@` or the tool fails fast.

### Example run scripts moved

The shell wrappers `run_align.example.sh`, `run_rpf.example.sh`, and `run_pipeline.example.sh` moved from the repo root to [examples/run_scripts/](../../examples/run_scripts/). The example YAMLs (`align_config.example.yaml`, `rpf_config.example.yaml`, `pipeline_config.example.yaml`) still live at the repo root.

## Migration

- **Existing pipelines**: code that hard-codes `plots_and_csv/` or `translation_profile/{p,a}/<sample>/...` paths must be updated. The new flat-layout paths are above. The `--plot_dir` flag still accepts a custom name; setting `--plot_dir plots_and_csv` continues to work.
- **`run_codon_correlation()` direct callers** (rare; users not going through the pipeline): the function now accepts a `site=` keyword (`"p"` or `"a"`) and reads `{site}_site_codon_usage_total.csv` rather than the old `codon_usage_total.csv` in the per-site parent.
- **`run_translation_profile_analysis()` direct callers**: pass `requested_sites=["p", "a"]` (or one of those) instead of `site_override=`. The legacy `site_override` keyword is still honoured for one cycle.
- **`run_coverage_profile_plots()` direct callers**: pass `requested_sites=` and the granular `write_read_coverage_raw=` / `write_read_coverage_rpm=` toggles. The legacy single `write_read_coverage=` keyword still works as a coarse toggle for both.

All 292 tests pass.
