# MitoRiboPy Release Notes: v0.4.1

Date: 2026-04-24

## Summary

This is a focused follow-up to v0.4.0 that fixes a real-world failure mode and consolidates the kit-preset list.

The v0.4.0 release added per-sample kit detection, but the failure response when no kit signature was found was too rigid — SRA-deposited / already-trimmed FASTQs failed with `adapter auto-detection found no known kit` because every kit's match rate sits at ~0%. v0.4.1 treats that universal-zero pattern as the strongest possible signal that the FASTQ is already trimmed, and falls through to a new `pretrimmed` kit so cutadapt skips the `-a` flag and only does length + quality filtering.

The kit-preset list also got noticeably long in v0.4.0 because we kept adding vendor-specific names (`truseq_smallrna`, `nebnext_smallrna`, `truseq_stranded_total`, `smarter_pico_v3`, `sequoia_express`, …) for kits that all share the same Illumina TruSeq Read 1 adapter. v0.4.1 consolidates them into adapter-family canonical names; old vendor names keep working as aliases.

## Highlights

- **`pretrimmed` kit preset**: cutadapt skips the `-a` flag entirely. `ResolvedKit.adapter` is now `str | None`; `_build_pass1_command` skips `-a` when adapter is `None`. Auto-inferred when adapter detection finds 0% match across every kit (the strongest signal that no adapter is there); also selectable explicitly via `--kit-preset pretrimmed`. Mixed batches with raw + pre-trimmed FASTQs in the same run resolve each sample independently.
- **Detection tuning flags**: `--adapter-detect-reads` (default 5000), `--adapter-detect-min-rate` (default 0.30), `--adapter-detect-min-len` (default 12), `--adapter-detect-pretrimmed-threshold` (default 0.05), and `--no-pretrimmed-inference`. All exposed in YAML too.
- **Kit-preset consolidation by adapter family** (with back-compat aliases). The canonical preset list:

  | Preset | 3' adapter | UMI |
  |---|---|---|
  | `auto` | per-sample detection | per-sample |
  | `pretrimmed` | none | none |
  | `custom` | user-supplied | configurable |
  | `illumina_smallrna` | TruSeq Small RNA | none |
  | `illumina_truseq` | TruSeq R1 | none |
  | `illumina_truseq_umi` | TruSeq R1 | 8 nt 5' |
  | `qiaseq_mirna` | QIAseq miRNA | 12 nt 3' |

  `illumina_truseq` covers most current Illumina-compatible total/stranded RNA preps in one preset: NEBNext Multiplex Small RNA, TruSeq Stranded Total RNA Gold, Takara SMARTer Stranded Total v3 Pico, Bio-Rad SEQuoia Express Standard, and any other prep on the standard Illumina R1 adapter without a UMI. `illumina_truseq_umi` covers NEBNext Ultra II UMI, Bio-Rad SEQuoia Complete UMI, etc.

  Old v0.4.0 vendor names (`truseq_smallrna`, `nebnext_smallrna`, `nebnext_ultra_umi`, plus the short-lived `truseq_stranded_total`, `smarter_pico_v3`, `sequoia_express`) are still accepted as aliases via `KIT_PRESET_ALIASES` and translated transparently. Existing YAML configs and CLI invocations need no changes.

## Changes worth noting on upgrade

- Adapter-detection failure with no fallback now falls through to the `pretrimmed` kit by default instead of hard-erroring. This is the fix for the `"adapter auto-detection found no known kit"` failure on SRA-deposited inputs reported in v0.4.0. Pass `--no-pretrimmed-inference` to keep the v0.4.0 strict behaviour.
- `DetectionResult` gains a `pretrimmed: bool` field. Code that constructs `DetectionResult` instances directly should pass the new field; the default is `False`, so existing callers keep their previous behaviour.
- 255 tests pass (15 in `test_align_sample_resolve.py` covering the new pretrimmed inference, mixed batches, and alias back-compat).
