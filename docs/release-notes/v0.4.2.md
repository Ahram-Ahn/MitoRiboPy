# MitoRiboPy v0.4.2 — Per-sample UMI overrides, output-layout refactor, strain rename

This release focuses on making mixed-library batches work cleanly out of the box, simplifying the output tree, and clarifying the public API around strains, codon tables, and renamed flags. PyPI install is now the recommended path.

## Highlights

### Per-sample UMI / kit overrides

Mixed-UMI batches no longer need a wrapping shell loop. An `align.samples:` YAML list (or a `--sample-overrides TSV` for hand-written configs) lets each sample declare its own `umi_length`, `umi_position`, `kit_preset`, `adapter`, and `dedup_strategy`:

```yaml
align:
  kit_preset: auto                     # global default
  fastq: input_data/seq
  samples:
    - name: sampleA
      kit_preset: illumina_truseq_umi
      umi_length: 8
      umi_position: 5p
    - name: sampleB
      kit_preset: qiaseq_mirna
      umi_length: 12
      umi_position: 3p
    - name: sampleC                    # SRA-deposited, already adapter-clipped
      kit_preset: pretrimmed
      umi_length: 0
```

`mitoribopy all` materialises the block as `<output>/align/sample_overrides.tsv` and feeds it to `mitoribopy align --sample-overrides`. Each per-sample override is reported in `kit_resolution.tsv` with a `source` column starting with `per_sample_override:` so the provenance file always tells the truth.

### Unified per-site output layout

The split `translation_profile_p/` / `translation_profile_a/` / `coverage_profile_plots_p/` / `coverage_profile_plots_a/` directories collapse into one nested tree per stage:

```
translation_profile/<sample>/{p,a}/{footprint_density, translating_frame, codon_usage}/
coverage_profile_plots/
    read_coverage_*/                # site-INDEPENDENT, written once
    {p,a}/site_density_*/           # site-DEPENDENT
```

The site subdir is always present, even in single-site runs, so downstream tooling never has to branch on `--analysis_sites`. The site-independent `read_coverage_*` plots are no longer regenerated for both sites — the orchestrator skips them on the second pass — roughly halving `coverage_profile_plots/` disk usage on the default `--analysis_sites=both` run.

The `p_site_coverage_*` inner directories are renamed to `site_density_*` because the parent dir already names the site.

### `footprint_density` CSV cleanup

Per-transcript `*_footprint_density.csv` columns are now `Position, Nucleotide, A_site, P_site`. Three changes:

- `E_site` dropped: it was always `P_site - 3` (a fixed ribosomal-geometry constant), redundant with the `P_site` column, and unread by any downstream consumer.
- Reordered to `A_site` before `P_site`: the A-site is what protein-synthesis analyses care about; `P_site` stays as the canonical reporting frame.
- `*_selected_depth` dropped: it was an outlier-capped duplicate of the primary site used only for the depth bar plot, which now caps in memory before plotting and never writes to disk.

### `debug_csv/` removed

The always-on `debug_csv/` subdirectory under each sample's translation-profile output is gone. The three CSVs it wrote (`read_cov_map`, `coverage_usage_per_sample`, `coverage_usage_a_site`) were intermediate dumps with no downstream consumer; their existence inflated the output tree on large transcriptomes.

### Strain rename to species names

The opaque short codes are gone in favour of self-documenting species names:

| Old | New |
|---|---|
| `-s h` | `-s h.sapiens` (default) |
| `-s y` | `-s s.cerevisiae` |
| `-s vm` / `-s ym` | dropped — use `-s custom` and pick a built-in NCBI Genetic Code with `--codon_table_name` |

`h` and `y` are still accepted as deprecated short synonyms (one `[mitoribopy] DEPRECATED:` line on first use, then continue) so existing pipeline_config.yaml files keep working.

For non-human / non-yeast organisms, the README has a new [Custom organisms](../../README.md#custom-organisms) section listing the most common NCBI Genetic Codes by organism group (mouse → `vertebrate_mitochondrial`, *Neurospora* / *Aspergillus* → `mold_mitochondrial`, *Drosophila* / *C. elegans* → `invertebrate_mitochondrial`, etc.), the annotation CSV schema, and a worked mouse mt example.

### New `short` footprint class

For mapping truncated RNase products that sit just below the canonical monosome window:

```bash
$ mitoribopy rpf --footprint_class short ...
# rpf=[16, 24], unfiltered_read_length_range=[10, 30]; same defaults across species.
```

`disome` ranges are also rebalanced to current literature: `h.sapiens` 50–70 (was 60–90), `s.cerevisiae` 60–90 (was 65–95). Disome unfiltered window is tightened to 40–100 nt to match.

### Per-sample resume markers

A 50-sample run that died at sample 30 can now be resumed without redoing the first 30. Every successful sample writes `<output>/.sample_done/<sample>.json` containing its full `SampleCounts` row. `mitoribopy align --resume` (auto-set by `mitoribopy all --resume` when `read_counts.tsv` is missing) reloads samples whose marker is present and only re-runs the rest. Markers are written atomically (`.tmp` then rename) so a kill mid-write cannot leave a half-flushed file that would silently feed the wrong row into the aggregated table.

### Intermediate file cleanup

By default the align stage now deletes regenerable intermediates as soon as the next step consumes them: `trimmed/<sample>.trimmed.fq.gz` (after contam-filter), `contam_filtered/<sample>.nocontam.fq.gz` (after mt-align), and the pre-MAPQ `aligned/<sample>.bam` (after MAPQ filter). On a typical 100M-read sample this saves ~3–4× the size of a single FASTQ in regenerable copies. Pass `--keep-intermediates` for debugging.

When the resolved `dedup_strategy` is `skip`, the orchestrator no longer writes a hardlink/copy of `mapq.bam` under `deduped/`. The downstream BED conversion reads BAM content (not paths), so feeding `mapq.bam` directly is identical and saves the duplicated `dedup.bam` + `dedup.bam.bai` pair on every no-UMI sample.

### Publication-quality codon-correlation plot

`mitoribopy rpf --cor_plot` now writes a publication-ready scatter under `codon_correlation/`:

- Square aspect with shared axis limits + identity (y = x) reference line so deviation from the diagonal is immediately visible.
- Okabe-Ito colour-blind-safe palette per Category; alpha + small marker edge to mute overplotting at the low-coverage corner.
- Compact `r / R² / p / slope / intercept / N` stat box.
- Top-10 highest-residual codons get **leader-line labels** placed greedily so labels no longer overlap each other or pile up on dense clusters.
- Saved as both SVG (vector, for figures) and 300 dpi PNG (for slides).

### Renamed CLI flags (with deprecated synonyms)

| Old | New |
|---|---|
| `--merge_density` | `--codon_density_window` (clearer name; the flag applies a ±1 nt smoothing window, not a frame collapse) |
| `--mrna_ref_patterns` | `--mt_mrna_substring_patterns` |
| `--offset_pick_reference selected_site` | `--offset_pick_reference reported_site` |

All old names emit a single `[mitoribopy] DEPRECATED:` line on first use and continue to work.

### PyPI install

```bash
$ python -m pip install mitoribopy
```

`mitoribopy` is published at [pypi.org/project/mitoribopy](https://pypi.org/project/mitoribopy/). Installing from source (`pip install -e .` after `git clone`) is still supported for users who need an unreleased fix.

### Diagrams + docs refresh

Four large-format diagrams (≥ 1920 px wide PNGs) replace the previous narrow Mermaid source under `docs/diagrams/`: an LR pipeline overview plus one stage-detail diagram per stage (align, rpf, rnaseq). Generated by `python docs/diagrams/render_diagrams.py` with matplotlib — no Node / mermaid-cli dependency.

The exhaustive `pipeline_config.example.yaml` at the repo root replaces the previous flat-dict `pipeline_config.example.json`. It shows every flag from each `--help` with its default value and a 1-line comment, so users can copy and edit instead of grepping the help output.

The `src/mitoribopy/utils/` empty subpackage was removed (it was an unused scaffold).

## Bug fixes

- **`align.trim`** — the two-pass 3' UMI flow (QIAseq miRNA and similar) used to leave the pass-1 minimum length at the bare `--min-length`. With a 12 nt UMI and the default `min-length=15`, that let pass 1 emit 15 nt reads, which became 3 nt inserts after pass 2 and silently inflated the short-read tail of the RPF length distribution. Pass 1 now uses `min_length + umi_length` so the post-pass-2 insert is guaranteed to meet the user-requested floor.
