# MitoRiboPy Release Notes: v0.3.0

Date: 2026-04-23

## Summary

This release expands MitoRiboPy from a Ribo-seq-only analysis package into a full mt-Ribo-seq pipeline that spans raw FASTQ all the way through translation-efficiency integration with paired RNA-seq. The CLI gains three new subcommands and an end-to-end orchestrator; the data model gains a strict reference-consistency gate so Ribo-seq and RNA-seq sides cannot drift.

## Highlights

- **New subcommand CLI** under `src/mitoribopy/cli/`:
  - `mitoribopy align` — FASTQ → BAM + BED6 + per-sample read counts (cutadapt + bowtie2 + umi_tools + pysam).
  - `mitoribopy rpf` — Ribo-seq analysis pipeline (the v0.2.x core, now an explicit subcommand).
  - `mitoribopy rnaseq` — DE-table integration; TE / ΔTE with a SHA256 reference-consistency gate.
  - `mitoribopy all` — end-to-end orchestrator with a composed `run_manifest.json`.
- **Adapter auto-detection** (`mitoribopy align --adapter-detection {auto|off|strict}`, default `auto`): scans the head of the first input FASTQ for known adapter signatures and either auto-selects the matching preset or hard-fails on mismatch (`strict` mode). Catches the silent failure where a wrong `--kit-preset` drops ~99% of reads as "too long" without any error.
- **`mitoribopy all --print-config-template`**: prints a fully-commented YAML covering every stage with sensible defaults and inline preset choices.
- **`--footprint_class {monosome,disome,custom}`** (default `monosome`): biological footprint preset that injects sensible defaults for `-rpf` and `--unfiltered_read_length_range`. Monosome defaults are h/vm 28-34, y/ym 37-41, unfiltered 15-50; disome widens to h/vm 60-90, y/ym 65-95, unfiltered 40-110. Explicit user values always win.
- **New strain presets `vm` (any vertebrate mito) and `ym` (any fungus with yeast-mito codon code)**: both inherit the codon table for their anchor strain but require user-supplied `--annotation_file` and an explicit `-rpf` range.
- **Frame-colored P-site plots at CDS nucleotide resolution**: each position colored by `(nt_pos - cds_start) % 3` using the Okabe-Ito colorblind-safe palette. Frame-0 dominance is the canonical mt-Ribo-seq QC signature; the three-color partition makes it visible at a glance.
- **RPF-window band on the per-sample read-length distribution plot**: shades the RPF window so the reader sees which lengths entered downstream analysis.
- **Defensive guards** for predictable failure modes: malformed BED rows (`end <= start`) dropped with a per-file WARNING; FASTA records with no matching annotation row trigger a loud WARNING; `--threads <= 0` raises `SystemExit`.
- **Config files** in JSON / YAML / TOML (auto-detected by suffix). YAML uses PyYAML; TOML uses stdlib `tomllib` on Py 3.11+ or the optional `tomli` fallback via the `toml-py310` extra.
- **`PipelineContext` convention**: every piece of derived state lives on the context, not on the parsed `argparse.Namespace`.
- New dependencies: `PyYAML>=6.0`, `pysam>=0.22`.

## Notes

- `--use_rna_seq` (the v0.2.x legacy RNA-seq ratio module) is **deprecated** in v0.3.0 and removed in v0.4.0. Use `mitoribopy rnaseq` for DE-based TE / ΔTE analysis with the SHA256 reference-consistency gate.
- Plain `mitoribopy <flags>` (no subcommand) still routes to `mitoribopy rpf <flags>` with a stderr `DEPRECATION` warning.
