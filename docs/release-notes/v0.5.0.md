# v0.5.0 — `mitoribopy rnaseq` is now from-FASTQ-by-default + four new TE / DE / QC plots

`mitoribopy rnaseq` is rewired so the **default flow takes raw RNA-seq + Ribo-seq FASTQs and a transcriptome FASTA** and runs trimming, alignment, counting, and pyDESeq2 itself before emitting TE, ΔTE, and a six-figure plot set. The original "bring your own DE table" flow stays fully supported as the **alternative flow** (`--de-table`); it is the right path for publication-grade DE statistics over the full transcriptome. The two flows are mutually exclusive.

## Added: `mitoribopy rnaseq` default flow (from-FASTQ)

### How it works

The default flow is selected by passing `--rna-fastq …`. The alternative flow stays available via `--de-table …` (mutually exclusive with the default). Running `mitoribopy rnaseq` with no flags now lists the default-flow required args in the missing-flag error message and emits a HINT line pointing at the alternative.

```bash
mitoribopy rnaseq \
  --rna-fastq input_data/rna_seq/   \
  --ribo-fastq input_data/ribo_seq/ \
  --reference-fasta references/human-mt-mRNA.fasta \
  --gene-id-convention bare         \
  --condition-map samples.tsv       \
  --condition-a control             \
  --condition-b knockdown           \
  --output results/rnaseq           \
  --align-threads 8
```

The subcommand:

1. **Detects SE vs PE per sample** from filename mate tokens (`_R1/_R2`, `_1/_2`, `.1./.2.`, `_R1_001/_R2_001`, `_read1/_read2`).
2. **Auto-detects the adapter** via the existing `align.adapter_detect` (single FASTQ scan against the kit-preset registry); falls back to `pretrimmed` when the scan is ambiguous.
3. **Auto-infers UMI presence** from per-position Shannon entropy on the first/last 16 nt of each read; conservatively returns `length=0` when the entropy distribution doesn't show a clear flat-vs-biological discontinuity (real Illumina reads are uniformly high-entropy across the entire 16-position window, so the conservative path is the common case).
4. **Builds (or reuses) a content-addressed bowtie2 index** at `<workdir>/bt2_cache/transcriptome_<sha12_of_fasta>`. Repeated runs against the same FASTA skip the rebuild. Pass `--bowtie2-index <prefix>` to skip the build entirely.
5. **Trims + aligns each sample.** SE path reuses `align.trim.run_cutadapt` and a local bowtie2 wrapper; PE path runs cutadapt with `-a` / `-A` and bowtie2 with `-1` / `-2`. PE + UMI is currently `NotImplementedError`.
6. **Counts reads per transcript** (primary mapped only, read1-only for paired records so each fragment contributes 1).
7. **Builds a sample sheet + runs pyDESeq2** on the RNA-side counts with the user-supplied contrast (`--condition-a` vs `--condition-b`). Result is serialised in the canonical `DeTable` DESeq2 schema and written to `<output>/de_table.tsv`.
8. **Falls through** into the existing TE / ΔTE / plot path. The reference-consistency hard-gate is skipped (we hash the FASTA ourselves and record the hash under `from_fastq.reference_checksum` in `run_settings.json`).

### Auto-pseudo-replicate for n=1 conditions

pyDESeq2 needs at least two samples per condition to estimate dispersion. Real-world experiments frequently have only one library per condition. To prevent a cryptic dispersion-fitter error, when a condition has exactly one sample its FASTQ is automatically stream-split by record parity (record N → `rep1` if even, `rep2` if odd) so pyDESeq2 sees n=2. Each split emits a stderr WARNING:

```text
[mitoribopy rnaseq] WARNING: RNA-seq condition 'KO' has only 1 sample
('KO_rnaseq'); auto-splitting reads by record parity into pseudo-
replicates 'KO_rnaseq_rep1' / 'KO_rnaseq_rep2'. These are mechanical
halves of the same library, NOT biological replicates — DESeq2
dispersion estimates will be artificially low. Pass
--no-auto-pseudo-replicate to disable.
```

The augmented condition map (original entries + rep1 / rep2 entries) is persisted to `<output>/condition_map.augmented.tsv`.

### New optional dependency: `[fastq]` extra

Mode B requires `pydeseq2 >= 0.4`. It is **soft-imported** so the existing pre-computed-DE flow keeps working without it. To enable Mode B:

```bash
pip install 'mitoribopy[fastq]'
```

## Added: four new TE / DE / QC plots

The plot directory was previously two figures (`mrna_vs_rpf.png` + `delta_te_volcano.png`). v0.5.0 expands it:

| Plot | What it shows | Available in |
|---|---|---|
| `ma.png` | log10(baseMean) vs log2FoldChange of the RNA-side DE; points coloured by `padj < 0.05`. | both flows |
| `te_bar_by_condition.png` | log2(TE) per gene, bars grouped by condition with SE error bars across replicates. The primary biological readout. | both flows |
| `te_heatmap.png` | gene × sample log2(TE) heatmap, RdBu colormap centred at 0, columns ordered by condition so replicates cluster. Cell-annotated when ≤ 200 cells. | both flows |
| `sample_pca.png` | PC1 vs PC2 from log1p counts, colour = condition, marker = assay. SVD-based, no scikit-learn dep. | default flow only (needs the full sample × gene counts matrix) |

The original two plots (`mrna_vs_rpf.png`, `delta_te_volcano.png`) are unchanged.

## Templates

Two new templates land in [`examples/templates/`](../../examples/templates/):

- [`rnaseq_config.example.yaml`](../../examples/templates/rnaseq_config.example.yaml) — flat YAML for `mitoribopy rnaseq --config <file>` covering both modes.

The two existing rnaseq-relevant templates were extended:

- [`pipeline_config.example.yaml`](../../examples/templates/pipeline_config.example.yaml) — the `rnaseq:` section now lists both flows' keys (default flow first, alternative second) with the mutual-exclusion rule and the auto-pseudo-replicate semantics documented inline.

## Migration

- **Existing `--de-table` users: no-op.** The alternative flow (`--de-table`, `--ribo-dir`, `--reference-gtf` / `--reference-checksum`) is unchanged at the behaviour level. Existing scripts and configs continue to work.
- **One small CLI behaviour change for users who previously ran `mitoribopy rnaseq` with no flags** (e.g. for help-string discovery): the missing-required-args error message now lists the **default-flow** required args (`--rna-fastq`, `--reference-fasta`, …) instead of the `--de-table` flow's args. The HINT line at the bottom of the error tells users about the alternative flow. If you scripted against the old error-message contents, update the assertion.
- **To use the default flow, install the extra:** `pip install 'mitoribopy[fastq]'`.
- **Output additions are additive.** The four new plots are emitted alongside the two existing ones in both flows; if your downstream tooling globs `<output>/plots/*.png` it now picks up the new files. Filename additions in `<output>/`: `de_table.tsv`, `rna_counts.tsv`, `rpf_counts.tsv` (long), `rpf_counts_matrix.tsv`, `condition_map.augmented.tsv` — all only present when running the default flow.
- **Templates rewritten to lead with the default flow.** `run_rnaseq.example.sh` and `run_pipeline.example.sh` default to `FLOW="default"` / `RNASEQ_FLOW="default"`; `rnaseq_config.example.yaml` and `pipeline_config.example.yaml` list the default-flow keys before the alternative-flow keys. The `Mode A` / `Mode B` labels used in earlier drafts are gone — we use **default flow** and **alternative flow (`--de-table`)** throughout.

## Known gaps

- **PE + UMI is `NotImplementedError`.** Preprocess UMIs into the read name first (e.g. via `umi_tools extract`), or pass `--de-table` with your own pre-computed DE results.
- **Genome / splice-aware aligner is out of scope.** Mode B builds a bowtie2 index from a transcriptome FASTA, matching the package's mt-mRNA focus.
- **DE statistics over the 13 mt-mRNAs are noisy.** Mode B is for exploratory pipeline runs and validation. For publication-grade DE, run DESeq2 / Xtail / Anota2Seq externally on the full transcriptome and use Mode A.
