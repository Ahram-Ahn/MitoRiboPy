# MitoRiboPy Release Notes: v0.2.0

Date: 2026-04-07

## Summary

This release turns MitoRiboPy into a cleaner standalone package for routine
analysis. It removes the runtime dependency on the packaged legacy pipeline,
moves built-in biological references into data files, and improves extensibility
for custom organisms.

## Highlights

- Removed runtime dependence on `src/mitoribopy/_legacy`.
- Added packaged JSON and CSV reference loading for codon tables and
  transcript annotations.
- Added custom-organism support with `--annotation_file`,
  `--codon_tables_file`, `--codon_table_name`, and `--start_codons`.
- Added custom reference templates under `examples/custom_reference/`.
- Added persistent run logging in `<output>/mitoribopy.log`.
- Split offset-selection bounds into separate 5' and 3' ranges.
- Added offset masking controls to suppress near-anchor enrichment artifacts.
- Renamed analysis and plotting modules to clearer public names such as
  `translation_profile_analysis`, `coverage_profile_plots`, and
  `structure_density_export`.
- Improved CLI help text and publishable README documentation.

## Upgrade notes

- The old `_legacy` package path is no longer part of the runtime pipeline.
- The legacy `--varna` flag is still accepted as a compatibility alias, but the
  public name is now `--structure_density`.
- Built-in annotations and codon tables are now loaded from packaged files under
  `src/mitoribopy/data/`.
