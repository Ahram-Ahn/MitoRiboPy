Validation Results: P-site vs A-site Behavior

1) Footprint selected-depth shift test (NT3 transcripts)
- Best shift across all 11 transcripts = -3 nt (A-site signal is downstream by 3 nt relative to P-site in this convention).
- This confirms expected codon register relationship.

2) Codon-usage output identity test
- NT3 codon_usage_total.csv (P-run) vs NT3 codon_usage_total.csv (A-run): not identical.
- Correlation exists but substantial differences are present, as expected when switching site.

3) Offset-table shift test
- Most read lengths show expected 5' offset shift pattern between p and a runs.
- One read-length (29 nt) shows a non-trivial chosen 5' offset difference due selection-range clipping effects when --min_offset 10 is enforced.

Why "small offset enrichment" can appear without a large stop/start spike in footprint plots
- In stop-aligned offset analysis, strong enrichment at small positive offsets (for example +4) is often from the 3' offset dimension (reads ending near stop codon), not necessarily from P-site position itself.
- With --offset_type 5, downstream footprint/codon analyses are driven by selected 5' offsets, not 3' offsets.
- Therefore, a large +4 signal in offset summary can coexist with modest/absent spike at stop codon in P-site footprint tracks.

Assessment of codon_data.py variable handoff to offset analysis
- Current handoff path is correct:
  - codon_data.py provides annotation lengths (l_tr, l_utr5, l_utr3).
  - main.py derives start_codon and stop_codon from these columns.
  - offset_analysis.py uses those derived start/stop coordinates directly.
- No missing assignment is observed in the current execution path for offset computation.

Residual Risks / Issues Found in Full Script Audit

High-priority scientific caveat
- IGV RPM normalization currently uses total reads aggregated across all references in read-count file, while IGV text describes normalization to mRNA-aligned reads.
- This can bias RPM scale if contaminants vary strongly between samples.
- Recommendation: optionally filter normalization denominator to mt_genome-aligned counts only.

Medium-priority engineering issues
- main.py: --downstream_dir argument is currently not consumed in run_pipeline.
- aligned_ratio.py: script executes code at import-time and depends on caas_jupyter_tools; should be guarded by if __name__ == "__main__" and decoupled from notebook-only dependency.

File-by-File Description (Top-level project files)

Python modules
- main.py
  - CLI entrypoint and orchestration of the entire workflow.
  - Parses arguments, loads annotations/config, runs read-length summaries, offset enrichment, selected offset table, in-frame/codon analysis, IGV plotting, optional VARNA/correlation/RNA-seq modules.

- pipeline_config.py
  - Defines DEFAULT_CONFIG and helper functions:
    - load_user_config(): read/validate JSON config keys.
    - resolve_rpf_range(): choose strain-specific default or user-specified RPF lengths.

- plot_style.py
  - Centralized matplotlib style defaults used by all plotting modules.

- bed_processing.py
  - BED I/O and preprocessing utilities:
    - compute_total_counts(): parse user read-count table and aggregate counts by sample.
    - process_bed_files(): length-filter BED reads and write per-length BED files.
    - compute_unfiltered_read_length_summary(): count unfiltered lengths and compute RPM normalization.
    - plot_unfiltered_read_length_heatmap(): heatmaps for count/RPM length distributions.

- offset_analysis.py
  - Computes codon-aligned offset enrichment and selected offsets:
    - compute_offsets(): per-read 5'/3' offsets relative to start/stop codons.
    - create_csv_for_offset_enrichment(): summary matrix by read length and offset.
    - determine_p_site_offsets(): pick most enriched offsets in selected range.
    - plot_offset_enrichment(): heatmap and line plots.

- inframe_analysis.py
  - Builds transcript-level A/P/E site depth maps and downstream translation-frame and codon analyses.
  - Writes footprint_density CSV/plots, frame usage CSV/plots, codon usage CSV/plots for primary selected site and A-site companion outputs.

- igv_style_plot.py
  - Produces multi-sample IGV-style per-transcript coverage plots:
    - read coverage and P-site density,
    - both RPM-scaled and raw-count panels,
    - global y-axis harmonization by transcript.

- varna_plot.py
  - Converts footprint density tracks into scaled values for VARNA coloring.
  - Outputs per-transcript txt/csv in sample-specific directories.

- codon_correlation.py
  - Pairwise codon-usage correlation between base sample and others.
  - For stop codons, values can be overridden from A-site output.
  - Produces all/masked CSV and scatter plots plus summary CSV.

- codon_data.py
  - Defines codon tables for standard, yeast mitochondrial, and human mitochondrial codes.
  - Stores transcript annotation tables for yeast/human (length and UTR metadata).

- rna_seq_analysis.py
  - Optional RNA-seq module.
  - Builds transcript coverage, summary tables, and optional Ribo/RNA ratio outputs using P-site footprints.

- subsample_bed.py
  - Utility script for reproducible random BED subsampling using reservoir sampling.
  - Useful for fast debug/testing with smaller read sets.

- aligned_ratio.py
  - Standalone enrichment plotting utility for read composition categories.
  - Not part of main pipeline execution path.

- __init__.py
  - Package metadata and exported module list.
