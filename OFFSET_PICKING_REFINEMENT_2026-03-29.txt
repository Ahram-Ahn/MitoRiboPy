Offset Picking Refinement Notes (2026-03-29)

Goal
- Make A-site and P-site selected 5' offsets stay codon-consistent (+3 shift for A-site relative to P-site) while keeping user-controlled offset range filtering.

Problem Observed (Old Behavior)
- Old policy selected offsets independently in the currently selected site space.
- With --min_offset 10 --max_offset 22, RL29 had:
  - P-site selected 5' offset = 22
  - A-site selected 5' offset = 11
  - delta = -11 (unexpected)
- Root cause: range clipping in A-space favored a different local peak than the P-space pick.

New Strategy Implemented
1) Canonical selection in P-site space (recommended policy)
- New CLI argument: --offset_pick_reference p_site|selected_site
- Default is p_site.
- Behavior:
  - If offset_site=p: select in p space directly.
  - If offset_site=a and offset_pick_reference=p_site:
    a) convert raw offsets back to p space
    b) apply min/max filtering and pick most enriched offsets
    c) convert selected offsets back to a space (+3 for 5', -3 for 3').

2) Deterministic tie-break for equal-count peaks
- Used inside determine_p_site_offsets:
  a) highest count
  b) highest local neighborhood support (offset-1, offset, offset+1)
  c) smallest absolute offset
  d) smallest signed offset

Code Locations
- offset_analysis.py
  - _convert_offsets_between_sites(...)
  - _pick_most_enriched_with_tiebreak(...)
  - determine_p_site_offsets(..., offset_site, selection_reference)
- main.py
  - new argument --offset_pick_reference
  - passed to determine_p_site_offsets
- pipeline_config.py / pipeline_config.example.json
  - default: "offset_pick_reference": "p_site"
- README.md
  - documented new option and recommended policy

Result
- 5' deltas are now consistently +3 across all read lengths.
- 3' deltas remain consistently -3.
- A-site/P-site footprint relationship remains codon-consistent (best shift -3 across transcripts in this comparison direction).

How to force legacy behavior
- Use: --offset_pick_reference selected_site

Recommended use
- Keep: --offset_pick_reference p_site
- Continue setting --min_offset/--max_offset as your biologically motivated range for canonical P-site geometry.
